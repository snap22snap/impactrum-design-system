Design System

Korea-based premium LED display 매뉴팩처러를 위한 디자인 시스템. 캔버스 한 장 위에 두 개의 옅은 off-white tone과 Wanted Sans Variable 한 가지 서체로 모든 페이지를 짜며, 카드는 자신이 올라간 page bg에 맞춰 자동으로 톤이 조정됩니다. Hierarchy는 size + letter-spacing이 만들고, 색면은 강조점에만 씁니다.

Foundations · 01

Color

네 개의 tier로 나뉩니다. Tier 1은 모든 action과 typography를 책임지는 monochrome chrome, Tier 2는 page bg와 card tint, Tier 3는 hairline, Tier 4는 강조점에만 쓰는 5색 accent.

Tier 1 · Monochrome chrome

--ink-*

Action layer. Primary CTA, h1/h2 headlines, wordmark, ribbon-badges는 모두 ink. 절대 배경으로 쓰지 않습니다.

--ink #111111 Primary CTA, headlines, wordmark
--ink-2 #2a2a2a Primary CTA hover
--ink-pressed #1a1a1a Primary CTA pressed
--ink-body #374151 Body text on canvas
--ink-mute #6b7280 Captions, secondary labels
--ink-soft #898989 Meta info, deemphasized
--ink-disabled #a1a1a1 Disabled primary text
--ink-disabled-2 #c4c4c4 Disabled secondary text

Tier 3 · Hairlines

--line-*

모든 border는 1px. 강조는 두께가 아니라 색 변화로.

--line #e5e7eb Default hairline
--line-strong #d4d4d4 White-button border, input border
--line-soft #f0f0ec Subtle dividers within cards

Tier 4 · Five-color accents

--accent-*

강조점에만. Line icon stroke, eyebrow text, footer column dot, pastel pill chip, inline link 강조. 절대 page bg나 card bg로 쓰지 않습니다.

Foundations · 02

Surface

Tier 2의 핵심 — 시스템의 중심 룰. Page background 3종 + card tint 6종(브라운·그레이 각 3개)으로 구성됩니다. 카드는 자신이 올라간 page bg에 맞춰 자동으로 톤이 조정됩니다.

Page backgrounds · Layer 1

--canvas, --surface-warm, --surface-cool

섹션 리듬을 만드는 3종. 모두 매우 옅은 톤 — 페이지가 attention을 가져가지 않습니다. Canvas가 default, warm은 따뜻한 섹션, cool은 technical/spec 섹션.

--canvas #ffffff Default · hero, prose-heavy
--surface-warm #fbfaf6 Featured, brand pages
--surface-cool #f5f6f9 Spec, technical

Adaptive card tints · Layer 2

.bg-* .card-*

시스템의 가장 새로운 룰. 카드는 같은 이름이지만 자신이 올라간 page bg에 따라 다른 hex를 갖습니다. Canvas 위 light 톤, warm/cool 위 한두 단계 짙은 톤. 디자이너/개발자는 hex를 외울 필요 없이 .bg-warm 안에 .card-brown을 쓰면 시스템이 알아서 #f7eee2를 골라줍니다.

Foundations · 03

Typography

Wanted Sans Variable 한 서체로 모든 type role을 처리합니다. Hierarchy는 size + letter-spacing이 만들고, weight는 500 → 600 → 700 단 세 단계. Display는 hero 한 페이지에 단 한 번만 — "큰 헤드라인"이 필요하면 기본은 h1(48px).

Weight axis

100-unit stepped

Wanted Sans는 "Variable" 이름과 달리 100 단위로만 변합니다. 540을 넣어도 500과 동일하게 렌더링됩니다. 한국어 본문은 400이 anemic하게 보여서 500이 기본입니다.

Foundations · 05

Iconography

시스템 아이콘은 line style, stroke 1.5px, 24×24 viewBox. Fill은 쓰지 않고 stroke만 사용 — 옅은 페이지 톤 위에서도 hairline border와 같은 리듬으로 호흡합니다. 기본 컬러는 --ink-body, accent zoning이 필요한 경우(섹션 eyebrow 옆, feature card icon 등)에만 5색 accent로 컬러링.

Components · 06

Components

모든 component는 정의된 token만으로 만들어집니다. Card는 page bg context에서 자동 톤 적응.

Buttons

.btn-primary, .btn-secondary, .btn-text

40px height, 8px radius, weight 600. State 변화는 색상 shift로만 — shadow나 transform은 쓰지 않습니다. Action layer는 monochrome — accent 색깔 버튼은 만들지 않습니다.

.btn-primary Default · Hover · Pressed · Disabled
.btn-secondary Default · Hover · Disabled
.btn-text Hover shows underline

Icon buttons

.btn + svg

아이콘은 16px, 1.5 stroke, gap 6px. Leading icon이 디폴트.

Pills & badges

.pill-orange, .pill-pink, .pill-violet, .pill-emerald, .pill-blue

24px height, 600 weight, 12px size, pill radius. Product type pill은 locked mapping — Indoor=orange, Outdoor=pink, Transparent=violet, Custom=emerald. Blue는 status·meta용 자유 사용.

FAQ accordionhairline-only

P1.5와 P1.2의 시각적 차이는 어디서 체감되나요? +

방송 카메라로 화면을 잡을 때 차이가 커집니다. P1.2는 카메라 노출 거리 2.5m부터 무모아레 영역이 시작되고, P1.5는 4m부터 시작됩니다. 일반 시청 거리에서는 두 픽셀 피치 모두 픽셀 인지 한계 바깥입니다.

설치 후 패널 교체는 얼마나 빠른가요? +

표준 캐비닛(320×160mm) 단위로 한 사람이 5분 이내에 교체 가능합니다.

투명 LED의 실제 투과율은? +

모델에 따라 65–85%. 매장 윈도우에서 가장 자주 쓰이는 TR-Glass 78은 78% — 유리 너머 매장 내부가 그대로 보입니다.

옥외 설치의 방진방수 등급은? +

OR 시리즈 전 모델 IP65/54 인증. 전면 IP65, 후면 IP54.

화면 컨트롤러는 별도 구매인가요? +

기본 구성에 NovaStar MCTRL-660 컨트롤러가 포함됩니다. 다른 컨트롤러 사용 시 옵션 변경 가능.

Editorial

화면이 끝나는 자리에서
공간이 시작된다

코엑스 미디어월은 2025년 6월에 P1.5 기반의 72㎡ 화면으로 리뉴얼되었습니다. 방송 카메라와의 거리에서 무모아레 임계점이 형성되도록 픽셀 피치와 refresh rate를 조합한 것이 이 프로젝트의 핵심 결정이었습니다.

P1.2를 선택하지 않은 이유는 비용이 아니라 시청 거리였습니다. 4m 이상에서 모아레가 사라지는 P1.5는, 카메라 워크를 인색하게 만들지 않으면서도 일반 관객의 픽셀 인지 한계 바깥에 있습니다. 기술 백서 ↗에서 측정 방식을 확인할 수 있습니다.

우리는 화면이 가장 잘 꺼지는 순간을 함께 설계합니다 — 상담 요청하시면 PM이 회신드립니다.

Patterns · 09

Usage Patterns

자주 틀리는 지점들을 do/don't로. 룰 자체는 짧지만, 잘못된 적용 사례를 보여주는 게 시스템 정착에 가장 도움이 됩니다.

Card hex 직접 쓰지 않기

.bg-* .card-* 패턴 사용

Card 배경은 hex로 hardcode 하지 말고, 항상 부모 page bg 클래스 + card 종류 클래스로. 그래야 카드가 어디에 옮겨가도 자동으로 적응합니다.

Do

<section class="bg-warm">
  <article class="card card-brown">
    <!-- system applies #f7eee2 -->
  </article>
</section>

Class만 쓰고 hex는 시스템이 골라줌. Page를 바꾸면 카드도 자동 적응.

Don't

<section style="background: #fbfaf6">
  <article style="background: #faf5ef">
    <!-- hardcoded, won't adapt -->
  </article>
</section>

Hex 박아놓으면 page bg 바꿔도 카드는 그대로 — adaptive 룰이 깨짐.

Display headline scale

.t-display once per page

72px display는 hero 단 한 번. "큰 헤드라인이 하나 더 필요하다" 싶으면 그건 h1(48px) — display가 아닙니다.

Do

화면이 공간을
깨운다

실내 공간의 화면

Section — .t-h1 48px

Display는 hero 한 번, 그 다음부터는 h1로 step down.

Don't

화면이 공간을
깨운다

Hero — .t-display 72px

실내 공간의 화면

Section — .t-display 72px ✗

Display를 두 번 쓰면 둘 다 평범해집니다 — 처음의 권위가 사라져요.

ink ≠ background

Ink는 typography와 primary CTA, ribbon-badge, wordmark에만. 섹션·카드 backdrop으로 쓰면 페이지 전체의 호흡이 깨집니다 — Impactrum은 "조명이 꺼진 미디어월" 같은 어두운 페이지를 만들지 않습니다.

Accent로 칠해진 버튼은 강조의 위계를 잃습니다 — 모든 게 강조면 어떤 것도 강조가 아닙니다.

Hierarchy via size, not weight

cap at 700

시각적 무게는 size + letter-spacing이 만듭니다. 작은 사이즈에서 weight를 800으로 올려도 한국어는 anemic하거나 답답해집니다.

Do

실내 공간의 화면

.t-h1 · 48px / 700 / -0.035em

사이즈가 만든 무게. -0.035em이 글자를 단단하게 묶어줍니다.

Don't

실내 공간의 화면

28px / 800 / 0 ✗

사이즈를 줄이고 weight를 800으로 올리면 한국어 자소가 막혀 보입니다 — 시각적 권위가 사라져요.

Korean letter-spacing

no negative on Korean text

Negative letter-spacing은 Latin 우세 헤드라인(≥28px)에만 사용합니다. 한국어 자소는 이미 wide한 자형이라 -0.04em이면 글자가 겹쳐 보입니다. .t-ko가 자동으로 letter-spacing을 0으로 해제합니다.

Do

실내 공간의 화면

Premium LED

.t-h1.t-ko (KR: 0) · .t-h1 (EN: -0.035em)

한국어 줄에는 letter-spacing 0, Latin 줄에는 -0.035em — 같은 .t-h1 안에서 분기.

Don't

실내 공간의 화면

48px / -0.04em on Korean ✗

자형이 겹치고 받침이 짓눌립니다.

Font smoothing

subpixel-antialiased only

-webkit-font-smoothing: subpixel-antialiased이 default. antialiased는 한국어 자소의 stroke를 얇게 만들어 본문이 anemic하게 보입니다. body에 한 번 선언하면 전체 페이지 일괄 적용.

Do

한글의 두께가 그대로 살아납니다

-webkit-font-smoothing: subpixel-antialiased;

자소가 본래 두께를 유지 — 본문이 단단하게 읽힙니다.

Don't

한글의 두께가 그대로 살아납니다

-webkit-font-smoothing: antialiased; ✗

자소 stroke가 얇아져서 weight 500이 마치 400처럼 — 한국어는 특히 민감합니다.

Section rhythm

canvas · warm · cool 흐름

세 page bg는 충분히 옅어서 인접 same-tone도 괜찮습니다. 다만 의도적인 톤 전환이 페이지에 "장"을 만들어주므로 — 보통 한 페이지에 canvas 2회, warm 2회, cool 1회 정도의 분배가 자연스럽습니다.

Do

01 · canvas — hero
02 · warm — features
03 · cool — specs
04 · canvas — stories
05 · warm — CTA / footer

의도적 전환이 페이지 호흡을 만듭니다.

Don't

01 · canvas
02 · pink block ✗
03 · emerald block ✗
04 · violet block ✗

Accent를 page bg로 칠하면 brochure 톤으로 미끄러집니다 — pill chip 안에서만 살아있어야 할 색입니다.

Accent zoning

where accent goes vs. not

Accent는 점(point)으로만. 페이지 안에서 accent가 살아있는 자리는 정해져 있습니다.

Yes · accent zones

  • ① Eyebrow text 컬러 (섹션 시작점)
  • ② Pastel pill / chip 의 배경+텍스트 페어
  • ③ Icon stroke (feature grid, status indicator)
  • ④ Footer column dot (locked mapping)
  • ⑤ Inline link 강조 — accent가 아니라 ink, 다만 underline의 hairline 강조

No · accent forbidden

  • ① Page bg / section bg
  • ② Card background
  • ③ Button background 또는 text
  • ④ Ribbon-badge (ink만)
  • ⑤ Headline 본문 컬러
