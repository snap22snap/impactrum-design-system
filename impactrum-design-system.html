Impactrum
Design System

Korea-based premium LED display 매뉴팩처러를 위한 디자인 시스템. 캔버스 한 장 위에 두 개의 옅은 off-white tone과 Wanted Sans Variable 한 가지 서체로 모든 페이지를 짜며, 카드는 자신이 올라간 page bg에 맞춰 자동으로 톤이 조정됩니다. Hierarchy는 size + letter-spacing이 만들고, 색면은 강조점에만 씁니다.

Foundations · 01

Color

네 개의 tier로 나뉩니다. Tier 1은 모든 action과 typography를 책임지는 monochrome chrome, Tier 2는 page bg와 card tint, Tier 3는 hairline, Tier 4는 강조점에만 쓰는 5색 accent.

Tier 1 · Monochrome chrome

--ink-*

Action layer. Primary CTA, h1/h2 headlines, wordmark, ribbon-badges는 모두 ink. 절대 배경으로 쓰지 않습니다.

--ink #111111 Primary CTA, headlines, wordmark
--ink-2 #2a2a2a Primary CTA hover
--ink-pressed #1a1a1a Primary CTA pressed
--ink-body #374151 Body text on canvas
--ink-mute #6b7280 Captions, secondary labels
--ink-soft #898989 Meta info, deemphasized
--ink-disabled #a1a1a1 Disabled primary text
--ink-disabled-2 #c4c4c4 Disabled secondary text

Tier 3 · Hairlines

--line-*

모든 border는 1px. 강조는 두께가 아니라 색 변화로.

--line #e5e7eb Default hairline
--line-strong #d4d4d4 White-button border, input border
--line-soft #f0f0ec Subtle dividers within cards

Tier 4 · Five-color accents

--accent-*

강조점에만. Line icon stroke, eyebrow text, footer column dot, pastel pill chip, inline link 강조. 절대 page bg나 card bg로 쓰지 않습니다.

--accent-orange #fb923c Indoor LED · locked
--accent-pink #ec4899 Outdoor LED · locked
--accent-violet #8b5cf6 Transparent LED · locked
--accent-emerald #34d399 Custom LED · locked
--accent-blue #3b82f6 Free use, status, time

Foundations · 02

Surface

Tier 2의 핵심 — 시스템의 중심 룰. Page background 3종 + card tint 6종(브라운·그레이 각 3개)으로 구성됩니다. 카드는 자신이 올라간 page bg에 맞춰 자동으로 톤이 조정됩니다.

Page backgrounds · Layer 1

--canvas, --surface-warm, --surface-cool

섹션 리듬을 만드는 3종. 모두 매우 옅은 톤 — 페이지가 attention을 가져가지 않습니다. Canvas가 default, warm은 따뜻한 섹션, cool은 technical/spec 섹션.

--canvas #ffffff Default · hero, prose-heavy
--surface-warm #fbfaf6 Featured, brand pages
--surface-cool #f5f6f9 Spec, technical

Adaptive card tints · Layer 2

.bg-* .card-*

시스템의 가장 새로운 룰. 카드는 같은 이름이지만 자신이 올라간 page bg에 따라 다른 hex를 갖습니다. Canvas 위 light 톤, warm/cool 위 한두 단계 짙은 톤. 디자이너/개발자는 hex를 외울 필요 없이 .bg-warm 안에 .card-brown을 쓰면 시스템이 알아서 #f7eee2를 골라줍니다.

Foundations · 03

Typography

Wanted Sans Variable 한 서체로 모든 type role을 처리합니다. Hierarchy는 size + letter-spacing이 만들고, weight는 500 → 600 → 700 단 세 단계. Display는 hero 한 페이지에 단 한 번만 — "큰 헤드라인"이 필요하면 기본은 h1(48px).

Weight axis

100-unit stepped

Wanted Sans는 "Variable" 이름과 달리 100 단위로만 변합니다. 540을 넣어도 500과 동일하게 렌더링됩니다. 한국어 본문은 400이 anemic하게 보여서 500이 기본입니다.

Components · 05

Components

모든 component는 정의된 token만으로 만들어집니다. Card는 page bg context에서 자동 톤 적응.

Buttons

.btn-primary, .btn-secondary, .btn-text

40px height, 8px radius, weight 600. State 변화는 색상 shift로만 — shadow나 transform은 쓰지 않습니다. Action layer는 monochrome — accent 색깔 버튼은 만들지 않습니다.

.btn-primary Default · Hover · Pressed · Disabled
.btn-secondary Default · Hover · Disabled
.btn-text Hover shows underline

Spec table

.spec-table, .spec-row

Label은 600, value는 500. Tabular numerals로 자동 정렬.

Pixel pitch P1.8 mm
Brightness 1,200 nit (max)
Refresh rate 3,840 Hz
Module size 320 × 160 mm
Power consumption avg. 280 W/㎡
Operating temperature −10°C ~ +40°C

Footer

--surface-warm + locked dot mapping

Footer는 surface-warm. 페이지 다른 부분과 같은 밝기 — 절대 어둡게 하지 않습니다. 컬럼 dot은 locked mapping을 따릅니다.

Patterns · 06

Usage Patterns

자주 틀리는 지점들을 do/don't로. 룰 자체는 짧지만, 잘못된 적용 사례를 보여주는 게 시스템 정착에 가장 도움이 됩니다.

Card hex 직접 쓰지 않기

.bg-* .card-* 패턴 사용

Card 배경은 hex로 hardcode 하지 말고, 항상 부모 page bg 클래스 + card 종류 클래스로. 그래야 카드가 어디에 옮겨가도 자동으로 적응합니다.

Do

<section class="bg-warm">
  <article class="card card-brown">
    <!-- system applies #f7eee2 -->
  </article>
</section>

Class만 쓰고 hex는 시스템이 골라줌. Page를 바꾸면 카드도 자동 적응.

Don't

<section style="background: #fbfaf6">
  <article style="background: #faf5ef">
    <!-- hardcoded, won't adapt -->
  </article>
</section>

Hex 박아놓으면 page bg 바꿔도 카드는 그대로 — adaptive 룰이 깨짐.

Display headline scale

.t-display once per page

72px display는 hero 단 한 번. "큰 헤드라인이 하나 더 필요하다" 싶으면 그건 h1(48px) — display가 아닙니다.

깨운다

화면이 공간을
깨운다

Hero — .t-display 72px

실내 공간의 화면

Section — .t-display 72px ✗

Display를 두 번 쓰면 둘 다 평범해집니다 — 처음의 권위가 사라져요.
